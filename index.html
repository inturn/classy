React styling. Plain and simple.

  1. Write a string containing CSS.
  2. Assign it to a CSS prop on your component class.
  3. Use the @Classy decorator... and profit!

Yup. That’s all it takes to tightly couple the presentation of your React component along with its structure/behaviors. Don’t believe it? Let’s take a look at some examples then, shall we? :)

Basic Usage

With just a few lines of code, Classy will automatically dump a <style> tag ubti document.head before your component mounts.

With just a few lines of code, Classy will automatically dump a <style> tag into document.head before your component mounts.

import React from 'react';

// Import Classy
import Classy from 'react-classy';

// Decorate your class
@Classy
export default class MyClass extends React.Component {
  ...
  // Assign a `css` prop
  css = `
    .my-component {
      background: blue
    }
  `
  ...
}

What about Radium / React Style / etc?

Most React styling solutions force you to use inline CSS. Although this is often touted as a good thing, there are some significant drawbacks.

  1. You will lose support for many useful CSS features.

    Child selectors, pseudo-selectors, pseudo-elements, media-queries, and animations. At best, certain solutions will attempt to shim some of these features back with JS. But those implementations are not as performant or expressive as they are in CSS.

  2. You will make it difficult for others to style your components.

    Other React devs who use your components in their project will basically be forced to look at rendered markup, and write some really awful CSS with tons of !important flags just to change a few styles. Or worse, they’ll force you into using some inflexible theming solution that doesn’t solve your needs (I’m looking at you, Material-UI).
